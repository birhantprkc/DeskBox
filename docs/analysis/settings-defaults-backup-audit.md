# DeskBox 设置、默认值、重置与备份审计

日期：2026-07-17

## 结论

当前设置页的主要问题不是选项数量本身，而是常用设置、专业参数、数据管理和诊断工具混在同一层级。继续增加风琴只能压缩页面高度，无法降低用户的理解成本。

建议固定为“一级导航 -> 二级概览页 -> 三级详情页”。风琴仅用于存在明确依赖关系的高级参数，不再作为主要的信息架构。

## 建议的信息架构

| 一级菜单 | 二级概览与三级页面 |
| --- | --- |
| 常规 | 语言、开机启动、新手引导直接展示 |
| 外观 | 主题与颜色直接展示；三级页面：窗口材质、显示密度、标题栏与窗口、显示/隐藏动画 |
| 胶囊模式 | 总开关直接展示；三级页面：收起行为、胶囊内容与隐私、过渡动画、单格自定义 |
| 文件格子 | 三级页面：文件显示、自动叠放、收纳与路径 |
| 功能格子 | 保持功能列表；随记、待办、音乐和天气本身作为三级页面 |
| 快捷与交互 | 三级页面：全局快捷键、悬浮操作、窗口与鼠标行为 |
| 诊断与维护 | 三级页面：备份与恢复、数据健康、兼容性诊断、重置 |
| 关于 | 版本、更新、项目地址、反馈和赞赏 |

随记和待办已经是“功能格子 -> 对应功能”的三级页面，不应继续增加第四级。页面内部应改用静态小标题划分“标签页”“编辑与列表”“提醒”“数据”等区域，最多保留一个高级风琴。

自动叠放页是当前比较合适的三级页面范例。设置主 XAML 已接近 2850 行，后续应逐步拆成独立页面组件，降低新增设置时发生语言刷新、默认值同步和可见性遗漏的概率。

## 页面设计规则

- 二级概览页只展示入口、当前状态摘要和箭头，例如“云母 · 圆角 · 细边框”。
- 三级页内三个以内的相关设置直接排列，不使用风琴。
- 只有“主开关关闭后整组冻结”或“高级参数依赖某个模式”时使用风琴。
- 危险操作不与普通开关混排，统一放在页面底部的危险操作区。
- 所有摘要使用本地化显示名称，并在切换语言后重新计算。
- 增加设置搜索，搜索结果直接跳转到对应三级页面和设置项。
- 更新已经位于“关于”内，“诊断与维护”不应继续使用“重置与更新”的旧标题。

## 新用户默认值审计

`AppSettings` 当前有 123 个顶层属性，`ApplyDefaultPreferences` 显式处理了 111 个。未重置的 12 个属性为：

- 系统或用户选择：`Language`、`AutoStart`、`FeatureWidgetEnabledStates`、`QuickCaptureEnabled`、`TodoEnabled`。
- 用户数据和运行状态：`Widgets`、`DeletedWidgetIds`、`RecentOrganizationHistory`、`LastQuickCaptureFileWidgetId`。
- 环境状态：`DefaultManagedStorageRootPath`、`HasCompletedOnboarding`、`LastUpdateCheckAt`。

这些属性大部分应当保留，没有发现近期新增顶层设置被直接漏掉。但当前没有机制要求每个新增属性显式声明“恢复默认、保留数据、保留系统状态”中的一种，未来仍然容易遗漏。

### 默认体验建议

1. 所有功能格子默认关闭，但应用首次启动时如果没有文件格子，会自动创建一个连接默认收纳目录的入门文件格子。这个兜底是合理的，应增加测试保证后续引导调整不会破坏它。
2. `AutoStart` 模型默认值为 `true`，界面实际读取 Windows 注册状态，两者可能不一致。系统状态应作为唯一事实来源。
3. 收纳拖入默认使用 `Move`，且当前规范化逻辑会强制改回 `Move`。对新用户风险较高，建议首次询问或默认复制。
4. 随记和待办默认展示十行，与紧凑定位不一致。建议新用户默认随记三行、待办两行。
5. 待办默认在普通视图展示已完成任务，数据增加后容易混乱。建议普通视图默认隐藏，保留“已完成”标签。
6. 胶囊、自动叠放和剪贴板采集默认关闭是合理的，属于用户主动开启的高级或隐私功能。

### 模型遗留

- `FocusClickedWidgetOnRaise` 当前没有实际读取方。
- `ManagedDropAction` 声明支持复制，但没有完整设置入口和消费链路，并被规范化为移动。

这两个字段应明确实现或移除，不能继续作为看似可配置的模型属性存在。

## 重置策略审计

当前“恢复默认偏好”会保留语言、开机启动、功能格子开关、格子实例、路径和用户数据，整体方向正确。

但单格覆盖的处理不一致：

- 会清除单格标题栏覆盖。
- 会保留单格胶囊行为、胶囊宽度与位置。
- 会保留文件叠放覆盖。

建议拆成明确的四类操作：

1. 恢复当前页面默认值。
2. 恢复全部全局偏好，不改变单格配置和用户数据。
3. 清除所有单格自定义，包括标题栏、胶囊尺寸与位置、叠放覆盖。
4. 清除内容或完全重置应用，独立确认并先提供备份。

`RestoreDefaultPreferencesAsync` 在调用统一默认逻辑后，又手工同步大量 ViewModel 属性，而项目已经存在 `ApplySettingsSnapshot()`。这会导致新增设置后磁盘值已恢复、界面仍显示旧值。应改为单一默认策略加统一快照同步。

## 备份范围

当前备份完整打包 `%LOCALAPPDATA%\DeskBox\data`。因此以下近期功能已经被包含：

- 胶囊全局参数、单格宽度、位置和元数据。
- 自动叠放规则、未匹配策略和单格配置。
- 随记与待办标签页、预览行数、Enter 行为和提醒设置。
- 材质、边框、圆角、密度、标题栏和动画设置。
- 随记、待办及 DeskBox 内部复制保存的附件。

以下内容不在备份内：

- 文件格子映射目录中的实际文件。
- 默认收纳目录中的实际文件。
- 仅关联路径的外部附件。
- Windows 开机启动注册状态。
- 资源管理器快速访问固定状态。
- 外部绝对路径在另一台电脑上的有效性。

因此当前功能更准确的产品名称是“设置、随记与待办备份”，不能暗示文件格子里的全部内容都会被备份。

## 备份和恢复风险

1. 自动保留七份每日快照、恢复前保留五份快照，但设置内不可见，也无法一键回退。
2. 启动恢复失败后仅记录日志，用户看不到结果，待恢复标记还可能在后续启动继续重试。
3. 恢复包没有强制要求 `settings.json`，也没有逐文件校验和。
4. 备份版本只接受 schema 1，没有正式迁移链路；旧程序恢复新程序备份可能忽略新字段。
5. 手动导出时应用仍可写入随记、待办和附件，可能产生跨文件时间点不一致的备份。
6. 缩略图和临时导出缓存也会进入备份，可能造成快照膨胀。

## 备份页面建议

“备份与恢复”三级页面应展示：

- 最近自动备份时间、大小和保留数量。
- 立即备份、恢复、打开备份目录。
- 导出前的“包含”和“不包含”清单。
- 恢复前的应用版本、数据类别、附件数量、不可用路径和需要重新映射的目录。

长期可以区分三种产品语义：

- 本机备份：用于故障恢复，保留位置和绝对路径。
- 导出偏好：只迁移外观、交互和功能设置。
- 迁移到新电脑：检查路径并提供目录重新映射，不自动打包用户的大量文件。

## 整改优先级

### P0 数据安全与一致性

- 明确备份包含和排除范围。
- 让启动恢复失败对用户可见，并避免无限重复恢复。
- 强制核心文件、逐文件大小与 SHA-256 校验。
- 建立备份格式兼容性规则和恢复版本提示。
- 统一恢复默认偏好的语义。
- 为所有新增 `AppSettings` 属性增加重置策略覆盖守卫。
- 增加最新设置、胶囊元数据、自动叠放规则和附件的完整备份往返测试。

### P1 设置层级

- 拆分外观、胶囊和维护三级页面。
- 整理随记、待办和天气页面的静态分组。
- 修复多语言摘要刷新和外显值一致性。

### P2 易用性

- 设置搜索。
- 自动快照管理。
- 当前页面恢复默认。
- 新用户起始格子和默认展示密度优化。

### P3 迁移能力

- 跨设备迁移。
- 路径映射向导。
- 备份格式迁移链路。

## P0 验收项

- 修改或删除任意备份文件后，恢复必须在替换当前数据前失败。
- 缺少 `settings.json` 的备份不能恢复。
- 启动恢复失败只尝试一次，保留当前数据，并向用户展示可理解的错误。
- “恢复全部全局偏好”不改变任何单格覆盖和用户内容。
- 每个 `AppSettings` 顶层属性必须出现在明确的重置策略中。
- 最新的胶囊、自动叠放、随记、待办和附件设置通过完整备份往返测试。
