# DeskBox：轻量级 Windows 桌面整理工具——功能说明书

> 当前版本：1.2.9 | 开发者：Tianyu Zhu | 开源协议：GPL-3.0
> 技术栈：WinUI 3 + .NET 10 + Windows App SDK 2.2
> 适用系统：Windows 11（64 位）

---

## 一、DeskBox 是什么

DeskBox 是一款 Windows 11 桌面整理工具。它不替换你的桌面，而是在原生桌面之上补一层整理能力——用轻量的桌面"格子"来收纳文件、映射文件夹、记待办、随手记东西、控制音乐和看天气。

核心理念可以一句话概括：**文件仍然是普通文件，格子只是帮你收纳和快速访问。** 你不会被绑死在 DeskBox 里——卸载它，你的文件还在原来的文件夹中，原封不动。

这与 Fences 等传统桌面整理工具的思路不同。Fences 倾向于"接管桌面"，把所有桌面图标纳入自己的管理体系；DeskBox 更克制，它只在桌面上放置若干独立的小窗口（格子），每个格子有明确的功能边界，桌面其余部分完全不受影响。

## 二、安装

### 安装包

DeskBox 提供两种分发渠道：

- **Direct 渠道**：从官网或 GitHub Releases 下载 Inno Setup 构建的 exe 安装包（如 `DeskBox_Setup_1.2.9_x64.exe`）。
- **Store 渠道**：通过 Microsoft Store 安装 MSIX 包。

两个渠道功能一致，差异仅在于打包方式和更新机制。Direct 渠道自带独立更新器，Store 渠道由商店管理更新。

### 安装过程

安装到当前用户目录（非 Program Files），**不需要管理员权限**。这是一个刻意的设计——以管理员权限运行的桌面工具会导致资源管理器的拖拽功能失效，因此 DeskBox 选择用户级安装。

安装时会自动检测两个运行时依赖：

- .NET 10 Desktop Runtime x64
- Windows App Runtime 2.2 x64

如果系统缺少，安装器会联网下载并安装。

覆盖安装会保留已有的设置、格子配置和收纳目录中的文件。如果旧版本装在 Program Files 下，安装器会自动迁移到用户目录。

### 卸载

卸载前会先停止运行中的 DeskBox，然后询问是否删除本地数据（位于 `%LocalAppData%\DeskBox`）。收纳目录中的用户文件不会被静默删除，涉及文件操作前会提示确认。

## 三、新用户引导

首次启动 DeskBox 时会弹出引导窗口（Mica 背景材质，1040×740），带你完成 6 步初始设置。每步都有进度圆点指示，可以随时跳过。

### 第 1 步：欢迎

品牌动效展示，简述 DeskBox 的定位——在桌面上放格子来整理文件和功能。

### 第 2 步：收纳目录与快速访问

这是最关键的一步。DeskBox 需要一个"收纳根路径"——所有"收纳格子"背后的真实文件夹都存放在这里。默认路径是 `%UserProfile%\DeskBox`，你可以改成任意位置。

> 💡 **理解收纳目录**：如果你创建了一个叫"工作文件"的收纳格子，DeskBox 会在收纳目录下创建一个 `工作文件` 子文件夹。你拖进格子的文件实际上被移动到了这个子文件夹里。在资源管理器中打开收纳目录，你就能看到所有格子对应的文件夹。

这一步还有一个开关：把收纳目录固定到资源管理器的"快速访问"。这样你在资源管理器左侧栏就能直接访问收纳目录，不用一层层翻路径。如果你后续修改了收纳路径，DeskBox 会提示你迁移已有的格子数据。

## 四、托盘菜单——日常控制中心

引导结束后，DeskBox 常驻系统托盘。你不和格子直接交互时，它安静地待在托盘区。

### 左键单击：显示/隐藏全部格子

智能判断当前状态：如果格子已经显示，且前台窗口是桌面、任务栏或 DeskBox 自身，则隐藏全部格子；否则唤起全部格子到顶层。

这个操作也可以用全局快捷键（默认 F7）触发。日常使用中最频繁的动作就是这个——需要格子时唤起，不需要时收起，桌面瞬间回到干净状态。

## 五、五种格子类型

格子是 DeskBox 的核心交互单元。每个格子是一个独立的桌面小窗口，有自己的标题栏、内容和操作菜单。当前共有五种类型。

> 📸 **[配图 8]** 五种格子的并列展示——一张图里同时展示文件格子、待办格子、随记格子、音乐格子、天气格子。

### 5.1 文件格子

文件格子是 DeskBox 最核心的格子类型，用于收纳和整理桌面文件。它有两种来源：

**收纳格子**

背后对应一个真实文件夹（位于收纳目录下）。当你把文件拖入格子时，文件会被移动（或复制，取决于设置）到这个文件夹中。文件从桌面上消失，进入了有序的收纳结构。

创建方式：托盘菜单 → 创建格子 → 选择文件格子。

**映射格子**

指向一个已有的文件夹（可以是任意位置，不限于收纳目录）。映射格子只读展示文件夹内容，不会移动任何文件。适合那些你不想移动但希望在桌面上快速访问的文件夹。

创建方式：托盘菜单 → 映射文件夹 → 选择目标文件夹。

> 💡 **收纳 vs 映射的区别**：收纳格子会移动文件，映射格子不会。收纳格子的文件夹在收纳目录下由 DeskBox 管理，映射格子的文件夹是你自己指定的任意路径。

**文件操作**

| 操作 | 方式 |
|------|------|
| 拖入文件 | 从资源管理器或桌面拖文件到格子，默认移动，可在设置中改为复制 |
| 拖出文件 | 从格子拖文件到资源管理器或其他位置 |
| 双击打开 | 双击文件项打开（可在设置中关闭此行为） |
| 右键菜单 | 复制、剪切、重命名、删除、在资源管理器中显示 |
| 多选 | 框选或 Ctrl/Shift 点击多选，支持批量复制/剪切/删除 |
| 排序 | 名称 / 大小 / 类型 / 修改日期 |
| 视图切换 | 图标视图 / 列表视图（列表视图可显示详情） |

> 📸 **[配图 10]** 文件格子右键菜单——展示排序、视图切换等菜单项。

### 5.2 待办格子

待办格子是一个轻量的任务管理工具，直接放在桌面上。

**列表视图**

以卡片列表展示任务。每张卡片最多显示 10 行文字，左侧有颜色标记条（8 种颜色可选）。支持以下交互：

- 拖动排序
- 拉框多选
- 点击进入详情页
- 颜色筛选（点击颜色标签筛选对应颜色的任务）

**详情页**

点击任意任务卡片，格子内展开该任务的完整详情页（不弹新窗口），可以编辑任务内容、设置截止日期、提醒、重复规则和附件。

> 📸 **[配图 11]** 待办格子截图——左侧卡片列表，右侧（或展开的）任务详情页。

**提醒机制**

到达提醒时间时，系统弹出通知。通知上有三个操作按钮：

- **完成**：标记任务完成
- **贪睡**：延后提醒，可选 10 分钟 / 30 分钟 / 1 小时 / 明天

**重复任务**

支持设置重复规则（每日/每周/每月等），完成后自动生成下一次任务。

**批量操作**

多选后可以批量：复制、删除、设置颜色、设置提醒、设置截止日期、设置重复规则。

**可配置项**

新建任务位置（顶部/底部）、默认筛选颜色、标签页样式（Pivot/分段按钮）、提醒偏移时间、是否在底部显示统计、删除前是否确认、是否显示已完成任务。

### 5.3 随记格子

随记格子是一个图文记录工具，介于便签和笔记之间。

**列表视图**

以卡片列表展示所有随记，卡片显示正文预览。支持普通列表和固定列表，两个列表都支持拖动排序（带动画）。默认视图可在设置中选择（记录/固定/最近）。

**编辑页**

点击任意随记卡片，进入专注编辑页——只保留正文，没有列表干扰。支持：

- 纸张材质样式（多种纸张可选）
- 添加/替换图片
- 图片预览与复制
- 固定/取消固定
- 显示创建时间

> 📸 **[配图 12]** 随记格子截图——展示卡片列表和不同纸张样式的随记卡片。

**剪贴板记录**

可配置自动记录最近剪贴板内容（文本/链接/图片），有数量上限（默认 30 条）。方便你随手保存复制的内容。

**批量操作**

多选后可以批量：复制、删除、固定/取消固定、设置纸张样式。

### 5.4 音乐格子

音乐格子是一个桌面音乐控制面板。注意：它本身不是播放器，而是对系统正在播放的音乐进行控制。

**功能**

- 播放控制：上一首 / 播放暂停 / 下一首
- 播放模式切换
- 系统音量调节
- 封面居中裁剪展示
- 歌名跑马灯（文字过长时滚动）
- 封面平滑切换动画

**自适应布局**

当格子宽度小于 180px 时，自动切换为封面优先的紧凑布局——只显示封面和基本控制，隐藏歌名等次要信息。

**氛围取色**

可开启"封面氛围"功能，根据当前播放歌曲的封面颜色，为格子背景生成柔和的色调，营造沉浸感。

**悬停动效**

鼠标悬停在封面上时有轻量的动效反馈。

> 📸 **[配图 13]** 音乐格子截图——展示封面、歌名、播放控制按钮和氛围取色背景。

> 💡 音乐格子和天气格子都是**单例格子**——全局只能创建一个。重复创建不会生成新的，而是聚焦到已有的那个。

### 5.5 天气格子

天气格子展示当前城市天气和预报。

**定位方式**

- 自动定位（基于 IP 地址）
- 手动搜索城市

**展示内容**

- 今日天气 + 7 天预报
- 可配置显示项：日出日落、UV 指数、降水概率、湿度、风速、气压
- 两种皮肤：标准 / 丰富
- 当格子尺寸小于 150×150 时，自动切换为小尺寸专用布局

**单位配置**

- 温度：摄氏 / 华氏
- 风速：km/h / m/s / mph

**刷新间隔**

可选 15 / 30 / 60 / 180 分钟自动刷新。

## 六、格子窗口交互

所有类型的格子共享一套统一的窗口交互行为。

### 标题栏

每个格子顶部有一个标题栏，显示格子名称。标题栏支持以下操作：

- **拖动移动**：按住标题栏拖动格子到任意位置
- **双击重命名**：双击标题栏进入重命名模式
- **右键菜单**：重命名 / 切换视图 / 排序 / 锁定位置 / 锁定大小 / 添加格子 / 更多 / 删除
- **悬停操作按钮**：鼠标移到标题栏时显示快捷操作按钮（可在设置中配置显示哪些按钮：锁定位置、锁定大小、添加、更多、删除）

### 调整大小

拖动格子边缘可以调整大小。调整过程中有**对齐引导线**——当格子边缘接近其他格子边缘或屏幕工作区边缘时（8px 阈值），会出现高亮对齐线，帮助你对齐格子。可在设置中开关此功能。

格子最小尺寸为 150×150。

### 位置锁定

可以锁定格子的位置和大小。锁定后无法拖动或调整，防止误操作。需要解锁后才能再次移动。

> 📸 **[配图 15]** 格子调整大小时的对齐引导线——展示两个格子边缘对齐时的高亮线效果。

### 层级管理

DeskBox 提供两种格子层级模式（设置 → 交互中配置）：

**动态模式（默认）**

格子默认位于桌面层（z-order 较低），不遮挡其他窗口。需要操作格子时，点击格子标题栏或按全局快捷键，格子会被提升到顶层。当格子失去焦点后，自动回落到桌面层。

这种模式的好处是格子平时不挡事，需要时才浮上来。

**桌面固定模式**

格子始终钉在桌面层，不会被提升到顶层。适合你希望格子始终作为桌面背景一部分的场景。

### 标题模式

可分别配置"显示类格子"（文件、天气）和"交互类格子"（待办、随记、音乐）的标题显示模式：

- **标准**：正常显示标题栏
- **紧凑**：缩小标题栏高度
- **覆盖**：标题栏覆盖在内容上方
- **隐藏**：完全不显示标题栏

### 多显示器支持

格子记录了创建时的显示器和位置锚点（左上/右上/左下/右下）。当分辨率变化或显示器拓扑发生变化时（如外接/断开显示器），格子会根据锚点自动恢复到合理位置。

## 八、如何用 DeskBox 整理桌面

这一节不是功能罗列，而是基于 DeskBox 的设计理念，给出实际的使用建议。

### 核心原则

DeskBox 的设计哲学是"不替换桌面，只补一层"。意思是：桌面只放当前正在用的东西，其他都收进格子；格子可以随时唤起和隐藏，文件始终是普通文件，始终在真实文件夹里，不会和 DeskBox 绑死。

### 推荐工作流

**第一步：按用途创建收纳格子**

打开托盘菜单 → 创建格子，为不同用途创建文件格子。比如：

- 📁 工作文件——日常工作文档
- 📁 下载临时——从浏览器下载的东西先丢这里
- 📁 截图——截图工具的默认保存目标
- 📁 项目素材——当前项目的图片、资料
- 📁 待处理——需要后续跟进的文件

每个格子背后是一个真实文件夹，文件拖进去就从桌面消失了，进入有序的收纳结构。

**第二步：对不想移动的文件夹用映射格子**

有些文件夹你不希望移动（比如项目目录、代码仓库），但又想在桌面上快速访问。用托盘菜单 → 映射文件夹，把它映射成桌面格子。映射格子只读展示，不碰原文件。

**第三步：放置功能格子**

根据需要开启功能格子：

- 在屏幕一角放一个**待办格子**，记录今天的任务清单。用颜色标记不同优先级，设截止日期和提醒。
- 放一个**随记格子**，随手记下临时想法、会议要点、复制的内容。不需要打开专门的笔记应用。
- 角落放一个**天气格子**，出门前看一眼。
- 放一个**音乐格子**，工作时控制播放不用切窗口。

**第四步：善用快捷键**

记住全局快捷键（默认 F7）。工作时按一下唤起全部格子，用完再按一下隐藏。桌面在"工作模式"（格子全开）和"干净模式"（格子全收）之间一键切换。

**第五步：收纳目录固定到快速访问**

确保收纳目录已经固定到资源管理器的"快速访问"（引导第 2 步或设置 → 维护中可以操作）。这样在资源管理器里也能直接访问格子里收纳的文件，不用通过 DeskBox。

> 📸 **[配图 25]** 整理后的桌面效果——展示一个"工作模式"下的桌面，几个收纳格子有序排列，角落有待办和天气格子。

### 注意事项

1. **不要把所有文件塞进一个格子**。格子是分类工具，一个格子对应一个用途。格子太多反而乱，建议 4-8 个文件格子。
2. **映射格子不会移动文件**。如果你希望文件从桌面消失并进入收纳结构，用收纳格子而不是映射格子。
3. **音乐和天气格子是单例**。全局只能各有一个，重复创建不会生成新的。
4. **卸载 DeskBox 不会丢文件**。收纳格子里的文件都在收纳目录的真实文件夹中，卸载 DeskBox 只是删掉程序和配置，文件原封不动。
5. **如果拖拽不工作**，去设置 → 维护 → 拖拽异常诊断，一键修复。常见原因是程序以管理员权限运行（DeskBox 默认用户级运行，一般不会出这个问题）。

## 九、技术概览

> 本节面向技术人员，普通用户可跳过。

### 技术栈

| 层面 | 技术 |
|------|------|
| UI 框架 | WinUI 3 + Windows App SDK 2.2 |
| 运行时 | .NET 10 |
| MVVM | CommunityToolkit.Mvvm（源生成器模式） |
| UI 控件 | CommunityToolkit.WinUI |
| 托盘图标 | H.NotifyIcon.WinUI |
| 打包 | Inno Setup（Direct）/ MSIX（Store） |
| 多语言 | 自研 LocalizationService（JSON 字典） |

### 架构

采用 MVVM 分层架构：

- **Models**（16 文件）：AppSettings、TodoItem、WidgetConfig、WeatherData 等数据模型
- **ViewModels**（10 文件）：各格子视图模型 + SettingsViewModel
- **Views**（14+ 文件）：WidgetWindow（拆成 DragDrop/Clipboard/Menus/Selection/Rename 等 9 个 partial class）、SettingsWindow、OnboardingWindow 等
- **Services**（~70 文件）：WidgetManager（拆成 Storage/TrayAnimation/ZOrder/FeatureWidgets 四个 partial）、文件操作、音乐会话、天气、待办提醒/重复、设置、主题、全局热键、应用更新等
- **Helpers**：Win32 互操作、Shell 上下文菜单、图标提取、剪贴板等

### 分发

通过 `DeskBoxDistribution` 编译变量切换 Direct / Store 渠道，控制捐赠二维码、更新机制等渠道差异。

### 项目规模

- 主应用约 145 个 C# 文件 + 12 个 XAML 文件
- 测试 29 个文件，覆盖核心服务
- 外部依赖仅 7 个 NuGet 包
