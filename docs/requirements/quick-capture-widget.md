# DeskBox 随记功能格子需求说明

状态：规划稿
创建日期：2026-06-20
适用版本：非 AI 第一阶段

## 1. 背景

DeskBox 现有文件格子已经解决了桌面文件收纳、文件夹映射、快捷拖拽上传等问题。新功能不应重复“临时文件收纳”能力，也不应改变现有文件格子的心智。

后续可以新增一个独立的功能格子，用于承接 Windows 用户日常工作中的轻内容：

- 刚复制的一段文字。
- 截图或复制的图片。
- 临时想到的一句话。
- 常用回复、地址、发票信息。
- 一条很轻的待办。
- 一个稍后要打开或发送的链接。

这个功能格子不是文件格子，也不是完整笔记软件、待办软件或剪贴板软件。它更像 Windows 桌面上的“随记”：复制过的、想到的、马上要用的内容，可以先放在这里，之后快速复制、拖出、固定或删除。

## 2. 产品定位

### 2.1 一句话定位

随记是 DeskBox 的轻内容功能格子，用于记录最近复制、手动输入和常用固定内容，并支持快速复制、拖出和轻待办处理。

### 2.2 与现有文件格子的关系

- 文件格子负责文件收纳、文件夹映射和文件拖拽上传。
- 随记负责文字、链接、图片、截图和轻待办等轻内容。
- F7 仍然只是临时置顶动作，用于在其他窗口遮挡时把常驻格子抬到前面。
- 随记不应替代文件格子，也不应引导用户把文件拖到这里做临时收纳。

### 2.3 设计原则

- 大众可理解：用户不需要学习“智能分类”“工作包”“标签系统”等概念。
- 功能克制：先解决复制、记录、固定、拿走四个动作。
- Windows 原生感：视觉、交互、菜单和状态遵循 Win11 / WinUI 风格。
- 智能藏在背后：识别只改变展示和可用动作，不强迫用户理解分类。
- 隐私优先：剪贴板记录默认本地保存，支持暂停、清空和数量限制。
- 桌面不变乱：默认一个功能格子，内部用视图切换，不铺开多个格子。

## 3. 用户价值

### 3.1 普通办公用户

- 复制地址、通知、订单号后，能在随记里找回。
- 截图后可以临时留在格子里，稍后发给别人。
- 常用信息固定后，不用反复翻微信聊天记录或文档。

### 3.2 客服、销售和运营

- 固定常用回复、报价说明、收货地址、发票信息。
- 临时保存客户发来的要求。
- 从固定页快速拖出或复制话术到微信、网页表单和办公软件。

### 3.3 学生和内容创作者

- 暂存网页段落、参考链接、截图和标题。
- 发布内容时快速拿出标签、文案和图片。
- 不需要打开完整笔记软件就能保存轻内容。

### 3.4 开发者和技术用户

- 暂存报错信息、命令、日志片段、截图和 Issue 链接。
- 固定常用命令或说明模板。
- 减少在剪贴板历史、浏览器、聊天窗口之间反复切换。

## 4. 核心概念

### 4.1 随记

随记是功能格子的名称，不是一个 Tab。用户看到的是一个常驻桌面格子。

### 4.2 视图

随记内部包含三个视图：

| 视图 | 用途 | 典型内容 |
| --- | --- | --- |
| 记录 | 用户主动保存的内容 | 笔记、轻待办、链接、截图 |
| 固定 | 长期复用的重要内容 | 常用回复、地址、发票信息 |
| 最近 | 最近剪贴板内容 | 复制的文字、链接、图片 |

### 4.3 内容项

内容项是随记里的最小单位。第一阶段支持：

- 文本。
- 链接。
- 图片。
- 轻待办。

文件不作为第一阶段重点。若用户复制的是文件路径或拖入文件，可以显示为文本或链接，不做文件收纳语义。

## 5. 页面结构

### 5.1 默认布局

默认只展示一个当前视图，避免三个区域同时展开造成拥挤。

```text
随记                                      ···

[ 记点什么... ]

记录

☐ 明天下午确认报价
客户说发票开普票
https://example.com
截图 14:32
```

默认视图：

- 第一次打开默认展示“记录”。
- 后续记住用户上次停留的视图。

### 5.2 Hover 标签切换

鼠标进入格子标题区域或顶部操作区域时，显示轻量标签切换条：

```text
随记                                      ···
[ 记录 12 ] [ 固定 5 ] [ 最近 8 ]

[ 记点什么... ]

内容列表...
```

交互要求：

- 标签条默认隐藏，鼠标移入后淡入。
- 鼠标悬停在某个标签 150-250ms 后切换视图。
- 点击标签也可以立即切换，并把它记为当前视图。
- 鼠标离开格子 500ms 后标签条淡出。
- 切换视图不应改变格子尺寸。
- 拖拽过程中不触发 hover 切换，避免误操作。

### 5.3 触控和键盘兜底

- 触控设备没有 hover，标签条应常显或点击标题后展开。
- 键盘焦点在格子内时，`Ctrl + Tab` 切换下一个视图。
- `Ctrl + Shift + Tab` 切换上一个视图。
- `Esc` 收起展开态或取消编辑。

## 6. 三个视图的功能

### 6.1 记录视图

定位：

用户主动保存的轻内容。

入口：

- 在顶部输入框输入一句话，按 Enter 保存。
- 从最近视图点击“保存到记录”。
- 从剪贴板内容弹出提示中保存。
- 拖入文字、图片或链接。

内容展示：

- 文本显示一到两行预览。
- 链接显示链接图标、标题或 URL。
- 图片显示小缩略图和时间。
- 轻待办显示勾选框。

主要动作：

- 复制。
- 拖出。
- 编辑。
- 固定。
- 标记为待办。
- 删除。

### 6.2 固定视图

定位：

长期复用或重要内容。

典型内容：

- 公司地址。
- 发票信息。
- 常用回复。
- 账号说明。
- 常用链接。
- 常用命令。

内容展示：

- 默认紧凑列表。
- 内容项可手动排序。
- 鼠标悬停时显示复制、编辑、取消固定。

主要动作：

- 点击复制。
- 拖出为纯文本或图片。
- 编辑。
- 取消固定。
- 删除。

### 6.3 最近视图

定位：

最近剪贴板历史，偏临时内容。

来源：

- 复制文字。
- 复制链接。
- 截图或复制图片。

内容展示：

- 按时间倒序。
- 显示复制时间。
- 显示内容类型图标。
- 最近内容数量上限可配置，默认建议 30 条。

主要动作：

- 复制。
- 拖出。
- 保存到记录。
- 固定。
- 删除单条。
- 清空最近。

隐私控制：

- 支持暂停记录剪贴板。
- 支持一键清空最近。
- 最近内容默认只保存在本地。
- 固定内容和最近内容分开，清空最近不删除固定内容。

## 7. 内容识别

第一阶段不接入 AI，只做本地规则识别。

### 7.1 链接识别

当内容是 URL 或主要包含 URL 时，显示为链接内容项。

动作：

- 打开链接。
- 复制 URL。
- 保存到记录。
- 固定。

### 7.2 图片识别

当剪贴板中包含图片或用户拖入图片时，显示为图片内容项。

动作：

- 复制图片。
- 拖出图片。
- 保存到记录。
- 固定。
- 删除。

### 7.3 轻待办识别

当文本包含明显动作词或时间词时，可以给出轻待办建议。

示例信号：

- 明天、今天、下午、下周。
- 记得、提醒、帮我。
- 发、确认、提交、上传、处理、联系、回复。

行为：

- 不强制自动变成待办。
- 可以在内容项旁显示一个小勾选入口。
- 用户点击后转为轻待办。

### 7.4 普通文本

无法可靠判断时，按普通文本处理。

原则：

- 识别错了也不影响使用。
- 默认展示和复制都保持原文。
- 类型可由用户手动调整。

## 8. 拖拽与复制

### 8.1 拖出

拖出行为：

- 文本拖出为纯文本。
- 链接拖出为 URL 或标题 + URL，默认 URL。
- 图片拖出为图片数据。
- 待办拖出为任务文本，默认不带内部状态。

### 8.2 拖入

支持拖入：

- 文字。
- 链接。
- 图片。

拖入规则：

- 拖入当前视图时，优先保存到当前视图。
- 拖入固定视图时，直接固定。
- 拖入最近视图时，不建议保存到最近，应该提示保存到记录。

### 8.3 点击复制

每条内容项的主动作应是复制。

交互：

- 单击内容项主体复制。
- 复制成功后显示轻量反馈，例如“已复制”。
- 悬停时显示更多操作按钮。

## 9. Win11 原生样式要求

### 9.1 总体风格

- 使用 WinUI 3 控件和交互习惯。
- 背景可使用 Mica / Acrylic 或与现有 DeskBox 格子一致的轻量半透明效果。
- 圆角、阴影、边框与现有格子统一。
- 跟随系统浅色、深色模式。
- 使用系统强调色突出当前标签和选中状态。

### 9.2 排版

- 字体使用系统 UI 字体。
- 标题紧凑，不使用大字号营销式标题。
- 内容列表密度要适合桌面小组件。
- 单条内容默认一到两行，超出省略，详情在展开或编辑态展示。

### 9.3 标签条

标签条不是传统大 TabView，而是轻量分段导航。

要求：

- 高度紧凑。
- 当前视图用强调色或轻背景提示。
- 未选标签低对比度显示。
- 显示数量徽标，例如“最近 8”。
- Hover 和点击都可切换。

### 9.4 内容项

内容项使用列表行或轻卡片，不使用大面积卡片墙。

每项包含：

- 类型图标。
- 内容预览。
- 时间或状态。
- 悬停操作。

操作按钮：

- 默认隐藏。
- 鼠标悬停行时出现。
- 常用操作可以是复制、固定、删除。
- 更多操作进入右键菜单或 `···` 菜单。

### 9.5 右键菜单

右键菜单应符合 WinUI 菜单风格。

建议菜单：

- 复制。
- 编辑。
- 固定 / 取消固定。
- 保存到记录。
- 标记为待办 / 取消待办。
- 删除。

## 10. 状态与设置

### 10.1 空状态

记录为空：

```text
输入一句话，或把文字、图片拖进来。
```

固定为空：

```text
固定常用回复、地址或链接。
```

最近为空：

```text
复制文字、链接或图片后会显示在这里。
```

### 10.2 隐私设置

设置项：

- 启用 / 暂停记录剪贴板。
- 最近内容保留数量。
- 退出后是否保留最近内容。
- 清空最近内容。
- 清空所有随记数据。

默认建议：

- 启用剪贴板记录前给用户明确提示。
- 最近内容默认保留 30 条。
- 图片可选择是否记录，默认开启但提供关闭。
- 不记录云端，不上传。

### 10.3 数据保存

建议保存位置：

```text
%LocalAppData%\DeskBox\data
```

建议数据：

- 记录内容。
- 固定内容。
- 最近内容。
- 当前视图。
- 剪贴板记录设置。

## 11. MVP 范围

### 11.1 P0

- 新增随记功能格子。
- 支持记录、固定、最近三个视图。
- 支持 hover 展示标签条并切换视图。
- 支持手动输入文本记录。
- 支持记录最近复制的文字和链接。
- 支持记录剪贴板图片并显示缩略图。
- 支持内容项复制。
- 支持内容项固定、取消固定。
- 支持删除单条内容。
- 支持清空最近。
- 支持本地持久化。
- 支持 F7 临时置顶时正常使用。

### 11.2 P1

- 支持拖入文字、链接、图片。
- 支持拖出文字和图片。
- 支持轻待办勾选。
- 支持简单待办识别建议。
- 支持固定内容手动排序。
- 支持更多右键菜单。

### 11.3 P2

- 支持更丰富的图片操作。
- 支持导出记录。
- 支持更细的隐私过滤。
- 支持 AI 辅助整理，但不进入第一阶段。

## 12. 验收标准

### 12.1 基础记录

- 用户可以在顶部输入框输入文字并保存到记录视图。
- 保存后内容项显示在当前视图列表中。
- 点击内容项可以复制原文。

### 12.2 最近剪贴板

- 用户复制一段文字后，最近视图出现该内容。
- 用户复制一个 URL 后，最近视图以链接形式展示。
- 用户截图或复制图片后，最近视图显示图片缩略图。
- 用户可以清空最近内容。

### 12.3 固定内容

- 用户可以把记录或最近内容固定。
- 固定内容出现在固定视图。
- 清空最近不删除固定内容。
- 固定内容可取消固定。

### 12.4 Hover 标签

- 鼠标移入格子顶部区域时标签条出现。
- 鼠标悬停在标签上可以切换视图。
- 鼠标离开后标签条收起。
- 拖拽过程中不会误触发标签切换。

### 12.5 Win11 风格

- 浅色、深色模式下均可读。
- 当前标签、悬停态、选中态符合系统视觉风格。
- 内容项操作按钮默认隐藏，悬停显示。
- 右键菜单使用 WinUI 风格。

## 13. 产品边界

第一阶段不要做：

- 多级标签系统。
- 完整待办管理。
- 日历和提醒。
- 知识库和富文本笔记。
- 文件收纳替代。
- 多个功能格子默认展开。
- AI 自动整理。

最重要的产品边界是：

```text
文件继续交给文件格子。
轻内容交给随记。
F7 只负责临时置顶。
```

## 14. 结论

随记功能格子应该像 Windows 官方会做的系统小工具：克制、清楚、低学习成本。

它的价值不是功能大而全，而是把 Windows 用户每天都会遇到的轻内容处理问题压缩成一个简单体验：

```text
复制过的，看得见。
想到的，记得下。
常用的，固定住。
需要时，拿得走。
```
