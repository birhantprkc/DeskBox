# DeskBox 1.2.1

## English

DeskBox 1.2.1 is a stability and polish release after the 1.2.0 architecture update. It focuses on monitor/DPI resilience, desktop widget title icons, settings defaults, and a few small interaction details that are important for daily use.

### Highlights

- Improved widget positioning across display topology changes, DPI changes, external display plug/unplug, and 1080p-to-4K display swaps.
- Added configurable desktop widget title icon modes: color, filled mono, line mono, hidden, and localized text label. Color is the default for new installs and restored default settings.
- Added title-style selection to the file widget blank-area context menu.
- Moved the widget title icon preference into Settings -> Appearance -> Window appearance details.
- Aligned new-user defaults, reset defaults, and invalid-value fallbacks for animation and title-icon settings. The default widget animation is now consistently `SlideFade`.
- Moved Quick Capture default-view normalization into the Quick Capture settings normalization path.
- Expanded automated coverage for settings defaults, title icon defaults, Quick Capture default-view normalization, and monitor-aware widget positioning.

### Notes For Testing

- Test overwrite install from 1.2.0 and confirm existing widgets, files, Quick Capture data, Todo data, settings, and managed storage content remain intact.
- On a dual-monitor setup, test unplugging/replugging an external display and changing between different scale factors or resolutions.
- Check Settings -> Appearance -> Window appearance details and verify the widget title icon modes switch correctly.
- In a file widget, right-click blank space and verify the title-style menu matches the title-bar menu behavior.
- Use Reset settings and verify the widget animation returns to `SlideFade` and title icons return to Color.

### Compatibility

- Target system: Windows 11.
- Runtime dependencies: .NET 8 Runtime x64 and Windows App Runtime 2.1.3 x64.
- Overwrite installs preserve existing app settings, widget configuration, Todo data, Quick Capture data, and managed storage content.

### 主要更新

- 优化多屏和 DPI 场景下的格子定位：显示器拓扑变化、缩放变化、外接屏插拔、1080p 更换 4K 显示器时，格子会重新恢复到当前可见屏幕区域内。
- 新增桌面格子标题图标模式：彩色、面性单色、线性单色、隐藏和多语言文字标签。新安装和恢复默认设置都默认使用彩色图标。
- 文件格子空白区域右键菜单新增标题样式选择。
- 将格子标题图标偏好收进「设置 -> 外观 -> 窗口外观细节」。
- 对齐新用户默认值、恢复默认值和无效值兜底逻辑：默认格子动效统一为 `SlideFade`，标题图标默认统一为彩色。
- 将随记默认视图归一化逻辑挪回随记设置归一化路径，避免后续维护混淆。
- 扩展自动化测试覆盖：设置默认值、标题图标默认值、随记默认视图归一化，以及多屏感知的格子定位。

### 测试建议

- 建议从 1.2.0 覆盖安装测试，确认已有格子、文件、随记、待办、设置和收纳目录内容都能保留。
- 如果有双屏环境，建议测试外接屏插拔，以及不同缩放比例、不同分辨率之间切换。
- 检查「设置 -> 外观 -> 窗口外观细节」，确认格子标题图标模式切换正常。
- 在文件格子空白区域右键，确认标题样式菜单和标题栏菜单行为一致。
- 使用恢复默认设置，确认格子动效恢复为 `SlideFade`，标题图标恢复为彩色。
